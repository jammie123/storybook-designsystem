# Status message

Prvky sloužící pro komunikaci mezi systémem a uživatelem. 

## Anatomie prvku

## Stavy

-   Informativní
-   Success
-   Error
-   Warning
    

  

## Typy oznámení

-   Inline message
-   Toast
    

## Obecné pokyny pro použití

-   Zkuste nepoužívat více než jednu výstražnou zprávu najednou. Jsou navrženy tak, aby upoutaly pozornost uživatelů, a pokud používáte více než jednoho najednou, může být matoucí, co je důležité.
    
-   Pokud potřebujete použít delší zprávu , doporučujeme použít také název . Uživatelé mají tendenci ignorovat delší texty, název by měl obsahovat shrnutí zprávy.
    
-   Informace uvnitř výstražné zprávy by měly být stručné . Pokud je třeba něco vysvětlit podrobněji, doporučuje se použít akční tlačítko a postupně zobrazovat obsah.

## Toast messages

Zobrazí krátkou informativní zprávu, která se objeví pouze na krátkou dobu.

Toasty jsou dočasná oznámení, takže jsou nejlepší pro krátké a jasné zprávy, nehodí se pro zobrazování složitých chybových zpráv v tom případě použijte inline message, které zůstane na obrazovce a nezmizí po časové prodlevě.
